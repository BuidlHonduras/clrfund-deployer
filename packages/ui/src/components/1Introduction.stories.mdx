# Welcome to Motion System

Motion System is a framework that helps you build next generation UI components and can be reaplaced with chakra via themeing.
Motion System is built on typescript, styled-system and framer-motion, the library focuses on making microinteractions easy and is a fork of rebass, which is awesome but ideologically very different with motion-system.

Dragons be Here

## Styling
Components are styled using prop definitions from the style-system API and a custom theme that conforms to the Tailwind Specification.
You can still use the components without a theme context and pass regular css values

#### Example

```jsx
import { Button } from "@motion-system/core"

export const MyComponent = () => (
  <Button
    onClick={() => {
      console.log("test1");
    }}

    px="7" // interpreted by our systemUI tailwind theme as 1.75rem
    py="5" // interpreted by our systemUI tailwind theme as 1.25rem
    bg="black" // interpreted by our systemUI tailwind theme as #000
    color="white" // interpreted by our systemUI tailwind theme as #fff
    minWidth="xs" // interpreted by our systemUI tailwind theme as 20rem

    fontFamily="sans.safe" // interpreted by our systemUI tailwind theme as "Inter, system-ui, sans-serif"
    fontSize="sm" // interpreted by our systemUI tailwind theme as 0.875rem
    lineHeight="loose" // interpreted by our systemUI tailwind theme as 2
    fontWeight="light" // interpreted by our systemUI tailwind theme as 300
    letterSpacing="tight" // interpreted by our systemUI tailwind theme as -0.025em

    borderStyle="solid"
    borderColor="blue.200" // interpreted by our systemUI tailwind theme as #bfdbfe
    borderWidth="2" // interpreted by our systemUI tailwind theme as 2px
    borderBottom="2px solid"
    borderBottomColor="blueGray.300" // interpreted by our systemUI tailwind theme as #93c5fd

    sx={{
      "&:hover": { // must be valid css object keys
        backgroundColor: "blueGrey.900", // interpreted by our systemUI tailwind theme as #0f172a !NOTE: gray or grey are both fine
      },
    }}
    {...args}>
    Custom Button
  </Button>
)
```
### Space

```jsx

// sets margin value of 1.75rem (`space['7']`)
<Box m="7" />

// sets margin value of `-1 * 1.75rem`
<Box m="-7" />

// sets a margin value of `16px`
<Box m='16px'/>

// sets margin `'auto'`
<Box m='auto' />

// sets margin 1.75rem (`space['7']`) on all viewports and 1.25rem (`space['5']`) from the first breakpoint and up
<Box m={[ 7, 5 ]} />
```

The space utility converts shorthand margin and padding props to margin and padding CSS declarations.

- Numbers from 0 to the length of `theme.space` are converted to values on the [space scale](#defaults).
- Negative values can be used for negative margins.
- Numbers greater than the length of the `theme.space` array are converted to raw pixel values.
- String values are passed as raw CSS values.
- And array values are converted into [responsive values][responsive-styles].

Margin and padding props follow a shorthand syntax for specifying direction.

| Prop                  | CSS Property                   |
| --------------------- | ------------------------------ |
| `margin`, `m`         | margin                         |
| `marginTop`, `mt`     | margin-top                     |
| `marginRight`, `mr`   | margin-right                   |
| `marginBottom`, `mb`  | margin-bottom                  |
| `marginLeft`, `ml`    | margin-left                    |
| `marginX`, `mx`       | margin-left and margin-right   |
| `marginY`, `my`       | margin-top and margin-bottom   |
| `padding`, `p`        | padding                        |
| `paddingTop`, `pt`    | padding-top                    |
| `paddingRight`, `pr`  | padding-right                  |
| `paddingBottom`, `pb` | padding-bottom                 |
| `paddingLeft`, `pl`   | padding-left                   |
| `paddingX`, `px`      | padding-left and padding-right |
| `paddingY`, `py`      | padding-top and padding-bottom |



As of v4.0.0, verbose margin and padding props (e.g. `margin`, `marginTop`) can also be used instead of the shorthand props.

## Color

```jsx
// examples
// picks the value defined in `theme.colors.blue`
<Box color='blue' />

// picks up a nested color value using dot notation
// `theme.colors.gray[0]`
<Box color='gray.0' />

// raw CSS color value
<Box color='#f00' />

// background colors
<Box bg='blue' />

// verbose prop
<Box backgroundColor='blue' />
```

The color utility parses a component's `color` and `bg` props and converts them into CSS declarations.
By default the raw value of the prop is returned.
Color palettes can be configured with the [ThemeProvider][theming] to use keys as prop values, with support for dot notation.

## Layout

```jsx
// examples

// width `50%`
<Box width={1/2} />

// width `256px`
<Box width={256} />

// width `'2em'`
<Box width='2em' />

// width `100%` on all viewports and `50%` from the smallest breakpoint and up
<Box width={[ 1, 1/2 ]} />

// width from `theme.sizes`
<Box width='medium' />

// display
<Box display='inline-block' />
<Box display={[ 'block', 'inline-block' ]} />

// maxWidth
<Box maxWidth={1024} />
<Box maxWidth={[ 768, null, null, 1024 ]} />

// minWidth
<Box minWidth={128} />
<Box minWidth={[ 96, 128 ]} />

// height
<Box height={64} />
<Box height={[ 48, 64 ]} />

// maxHeight
<Box maxHeight={512} />
<Box maxHeight={[ 384, 512 ]} />

// minHeight
<Box minHeight={512} />
<Box minHeight={[ 384, 512 ]} />

// size (width & height)
<Box size={32} />
<Box size={[ 32, 48 ]} />

// overflow
<Box overflow='hidden' />

// overflowX
<Box overflowX='hidden' />

// overflowY
<Box overflowY='hidden' />
```

The layout utility includes style props for `width`, `height`, `display`, `minWidth`, `minHeight`, `maxWidth`, `maxHeight`, `size`, `verticalAlign`, `overflow`, `overflowX`, and `overflowY`.

The `width` prop is transformed based on the following:

- Numbers from 0-1 are converted to percentage widths.
- Numbers greater than 1 are converted to pixel values.
- String values are passed as raw CSS values.
- And arrays are converted to [responsive width styles][responsive-styles].
- If `theme.sizes` is defined, the `width` prop will attempt to pick up values from the theme

# Extending
This stuff might be useful if you are trying to build components ontop of this library. Hope you know what you're doing!

## Compose

The `compose` utility is used to combine multiple style functions together into one.
This utility can help improve performance when using multiple style props functions on the same component.

```js
import styled from '@emotion/styled'
import { compose, typography, space, color } from 'styled-system'

export const Text = styled('div')(
  compose(
    typography,
    space,
    color
  )
)
```


### propTypes

Prop type definitions are available for each style function to add to your component's propTypes object.
Each value in `propTypes` is an object which should be assigned (or spread) to the component's `propTypes`.

```jsx
import styled from '@emotion/styled'
import { width } from 'styled-system'

const Box = styled.div`
  ${width}
`

Box.propTypes = {
  ...width.propTypes,
}
```



## Variant

Creates a custom style utility to apply complex styles based on a single prop.

```js
import styled from '@emotion/styled'
import { variant } from 'styled-system'

const Card = styled.div`
  ${variant({
    variants: {
      normal: {
        p: 2,
        boxShadow: 'default',
        borderRadius: 2,
      },
      large: {
        p: 3,
        boxShadow: 'large',
        borderRadius: 4,
      },
    }
  })}
`
Card.defaultProps = {
  variant: 'normal',
}
// <Card variant='large' />
```

## Legacy Variants

The legacy variants require styles to be defined in the theme object and do *not* use `@styled-system/css` for transformation.

```js
import { textStyle, colorStyle, buttonStyle } from 'styled-system'
```

```jsx
// textStyle
<Text textStyle='caps' />

// colorStyle
<Box colors='warning' />

// buttonStyle
<Button variant='primary' />
```

## Defaults

Some style props include default, fallback scales if not defined in the `theme` object.

```js
// Default Breakpoints
const breakpoints = ['40em', '52em', '64em']
// @media screen and (min-width: 40em)
// @media screen and (min-width: 52em)
// @media screen and (min-width: 64em)

// default fontSizes
const fontSizes = [12, 14, 16, 20, 24, 32, 48, 64, 72]

// default space for margin and padding
const space = [0, 4, 8, 16, 32, 64, 128, 256, 512]
```
